لن تخسر المشتركين أبداً! 🎉

## 💾 نظام النسخ الاحتياطي

### كيف يعمل؟

1. **تلقائياً:** عند كل عملية حفظ، يتم إنشاء نسخة احتياطية
2. **محلياً:** يتم الحفظ في مجلد `backups/`
3. **متعدد:** الاحتفاظ بآخر 10 نسخ + نسخة "latest"
4. **آمن:** استرجاع تلقائي إذا فُقد الملف الأساسي

### كيف أحصل على نسخة احتياطية؟

استخدم أمر `/backup` في البوت (كأدمن):

```
/backup
```

سيرسل لك البوت ملف JSON يحتوي على جميع معرفات المشتركين.

**احفظ هذا الملف في مكان آمن!**

### كيف أسترجع نسخة احتياطية؟

#### تلقائياً (الأسهل!)

البوت يسترجع تلقائياً من آخر نسخة إذا فُقد الملف الأساسي.

#### يدوياً

```bash
python restore_backup.py
```

أو ضع ملف النسخة الاحتياطية في `data/users.json` وأعد تشغيل البوت.

## 🔄 التحديث من النسخة القديمة

إذا كان البوت يعمل بالفعل:

1. **استبدل `main.py`** بالنسخة الجديدة
2. **حدّث `.gitignore`** بالنسخة الجديدة
3. **ارفع على GitHub:**
   ```bash
   git add main.py .gitignore restore_backup.py BACKUP_SYSTEM_GUIDE.md
   git commit -m "Add automatic backup system"
   git push
   ```
4. **Railway سيعيد النشر تلقائياً**
5. **لا حاجة لتغيير المتغيرات البيئية**

## ⚠️ ملاحظات مهمة

### Railway والملفات المؤقتة

Railway قد يحذف الملفات عند إعادة النشر. لذلك:

✅ **استخدم `/backup` بشكل دوري** (أسبوعياً مثلاً)

### الخصوصية

- الملفات تحتوي فقط على معرفات المشتركين (أرقام)
- لا تحتوي على أسماء أو معلومات شخصية
- مع ذلك، احفظها في مكان آمن
